## C. Devpost written description (final prose)

**Trace — a design-decision memory agent for AEC.**

**Inspiration.** In construction, the design brief evolves across dozens of meetings, but the industry records *what* was decided, not *why*. Rationale lives in email and people's heads — and "nobody who's there at the beginning of a project is involved at the end." In Singapore the stakes are personal: under the Qualified Person regime (Building Control Act s.9), a *named* architect or engineer is **personally and criminally** liable for design decisions — yet the only record the law prescribes is structural-only, deviation-only paper, with no way to see when a later decision quietly falsifies an earlier one's premise. The Toh Guan Road fire — one life lost, 40 buildings later found with potentially non-compliant cladding — is what that gap costs. Trace is the missing memory.

**What it does.** Trace ingests design-meeting transcripts and (1) **captures** every decision as a structured record — decision + rationale + the assumptions it rests on + author + timestamp; (2) **actively fires an alert** the moment a new decision falsifies the premise an earlier one relied on (the temporal analogue of clash detection); (3) **recalls** the currently-valid critical decisions on demand, within a token budget, and honestly abstains when nothing is on record; and (4) **never deletes** — superseded decisions are invalidated, not erased, and every write is appended to a **SHA-256 hash-chained audit log** (`verify_audit_chain()`), making the "who changed what, when, why" trail tamper-evident — the defensible record a liable QP has never had, as code.

**How we built it.** A bi-temporal decision store on SQLite (valid-time + transaction-time, never-delete supersession, à la Graphiti). Invalidation is a deterministic AEC rule-pack gate — four SCDF rules, so the alert never mis-fires — paired with an **LLM premise check** that reads each prior decision's *stored assumptions* when the rules are silent. A **decision court** of three Qwen roles argues the conflict, writes the verdict's reasoning, and persists it to the record. Capture and recall use Qwen (qwen-plus) via function-calling on the Singapore Qwen Cloud endpoint, and the four core functions are exposed as **custom tools through Qwen-Agent** — the agent calls them itself. We also ship a **real MCP server** (official `mcp` SDK, stdio): eight **deterministic, keyless** tools over the Model Context Protocol — the never-delete record, bi-temporal time-travel, the fire-code `check_compliance` gate (returns the clause + official link), and `verify_audit_chain` — so any MCP client (Claude Desktop, a Qwen agent, an IDE) grounds on Trace's *certain* half with no key and no network. Recall is **hybrid** — lexical overlap blended with a **Qwen text-embedding** semantic signal, so a paraphrase with no shared words is still recalled; it degrades to the deterministic lexical path with no key. Court verdicts are **real state transitions** (a rejected proposal is `rejected` on the record; an allowed one is adopted and supersedes what it displaced), a second Singapore rule-pack (**BCA Periodic Façade Inspection**, `rules/sg-bca/`) proves packs are pluggable per code authority, and a **kb/ second-brain vault** (Obsidian-compatible markdown, wikilinked) ingests dropped notes as audit-chained episodes and projects every decision back as a note. 147 tests, TDD throughout.

**Challenges.** Keeping the demo's centrepiece deterministic (rule-pack-gated) so it's reliable on camera; separating *capture* (compile, don't retrieve) from *retrieval-to-budget*; and grounding the Singapore regulatory claims against primary sources (SCDF Fire Code Cl 3.5, Building Control Act s.9).

**Accomplishments.** A working four-part loop end-to-end on a real Qwen stack, an agent that calls its own Qwen-Agent tools, a **keyless MCP server** exposing the same deterministic core over the protocol, and a problem framing a *regulator already mandates*.

**What's next.** An ambient desktop bubble ("automation, not a tool"), per-app integration (a Revit add-in), and an on-prem open-weight-Qwen deployment so nothing leaves the firm's server.

**Built with:** Qwen (qwen-plus) · Qwen-Agent · MCP (official-SDK server, stdio) · Qwen Cloud (Singapore / DashScope) · Python · SQLite. Open-source (MIT).
