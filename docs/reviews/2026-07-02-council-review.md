# The Council's Verdict on Trace

## 1. Executive Verdict

Trace is a top-quartile hackathon entry with a bottom-quartile relationship between its pitch and its code. The concept — a bi-temporal, never-delete design-decision memory for AEC with active premise-invalidation, wrapped in a legally accurate Singapore QP/golden-thread framing — is the strongest Track 1 positioning the council has seen, and the deterministic demo engineering is genuine craft. But the fact-checks falsified the project's flagship technical claim (the "bi-temporal" as-of query returns an empty set under retroactive supersession, the norm in AEC practice), proved the documented install path fails on a fresh clone (four undeclared dependencies), and confirmed that judge-facing materials assert three components that do not exist (hybrid retrieval, an LLM premise check, MCP protocol integration). Every defect found is fixable within the five remaining days, and the internal documentation is honest enough to prove the team can do it — but as of 2 July, a judge who clones the repo and reads the Devpost draft meets a broken installer and a pitch the code falsifies. **Final overall score: 6.5/10** — contends for the track only if the remaining time is spent on truth and installability, not features.

## 3. Where the Council Agreed

All five members, with fact-check confirmation, converged on the following:

1. **The deterministic demo engineering is real craft.** The invalidation alert and court verdict are gated by the YAML rule-pack before any LLM call, and the abstention path provably never touches the LLM (`client=object()` tests). Even Marcus, the harshest scorer, called this "disciplined code."
2. **The internal documentation is unusually honest.** docs/06's assumption ledger, the [verified]/[inference] tagging, `.research/verification.md` contradicting the team's own researchers, and the self-labelled "AMBIENT (staged)" CLI card are genuine epistemic discipline, not decoration.
3. **The judge-facing materials overclaim.** README/docs/10 assert hybrid retrieval, an "LLM premise check," "MCP tools," and wrong test counts — all confirmed absent or wrong by fact-check and by this bench's own inspection of docs/10-pitch-kit.md:31,45,49.
4. **The invalidation engine is one scenario deep.** One YAML rule (under-encoding its own cited clause — Ingrid's find, conceded by everyone), a hardcoded 95m building, one plumbed attribute, and a positional `same_discipline[-1]` "breaks" heuristic that never reads the stored assumptions the product is named after.
5. **The fresh-clone experience fails.** requirements.txt omits pyyaml plus the numpy/soundfile/python-dateutil that qwen-agent needs; the README's own pytest command dies at collection (fact-checked in a clean venv).
6. **Marcus's five reproduced defects are real** — retroactive as-of hole, double-supersede fork, candidates=0 abstention falsehood, timestamp format fragility, COUNT(*)-based ID collision — and none has test coverage.
